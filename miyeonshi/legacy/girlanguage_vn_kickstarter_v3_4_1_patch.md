## PATCH 1. 세이브 슬롯 구성 + settings 전역 규칙  
**삽입 위치:** PART D > D6 세이브 스키마 바로 위

### 세이브 슬롯 구성
- 수동 슬롯: **3개**
- 오토 슬롯:
  - `autosave_ch1` ~ `autosave_ch5`
- 오토 세이브는 **최근 1개만 유지** (챕터 재진입 시 덮어쓰기)
- 오토 세이브는 **챕터 시작 시점 상태**를 기준으로 한다.

### settings 전역 저장
- settings는 **슬롯과 무관하게 전역 저장**
- 저장 위치: `localStorage.settings`

```json
{
  "typing": false,
  "typingSpeed": 20,
  "fontSize": "M"
}
```

## PATCH 2. 조건부 분기 최소 수량 규칙  
**삽입 위치:** PART D > D4 if[] 오버라이드 규칙 아래

### 분기 품질 하한선 (필수)
- **챕터당 최소**
  - 조건부 `say (if[])` : **4개 이상**
  - 조건부 `nextIf` : **2개 이상**
- Ch4(갈등 챕터)는 위 수치를 **상회하도록 권장**

이 규칙은 시나리오 리뷰 시 **통과/실패 기준**으로 사용한다.

## PATCH 4. chat 노드 운영 규칙 명시  
**삽입 위치:** PART E > chat 노드 샘플 아래

### chat 노드 운영 규칙 (확정)
- chat 노드는 **연출 전용**
- chat 내부에서:
  - `if[]` 사용 ❌
  - `choice` 사용 ❌
- 선택이 필요한 경우:
  - **반드시 chat → choice → chat** 구조로 분리
- 메시지는 `delay` 값에 따라 **순차 출력**

## PATCH 8. validate_scenario.js 스펙  
**삽입 위치:** PART H 하단

## H2. 시나리오 정적 검증 도구

### 기능
- 모든 `next` / `jump` 대상 존재 여부 검사
- 도달 불가능 노드 탐지
- 무한 루프 탐지 (선택지 없는 순환)
- BFS 기반 변수 범위 계산
  - trust ≥ 7 도달 가능 여부
  - fatigue ≥ 8 도달 가능 여부
- 엔딩 도달 가능성 검사

### 실행
```bash
node tools/validate_scenario.js
```

- 오류 존재 시 exit code = 1
- 경고만 있을 경우 exit code = 0
