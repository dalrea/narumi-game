# PART A. 캐릭터 & 톤 (서사 기준)

## A1. 캐릭터 바이블

### 히로인: 미나 (ID: `mina`)
- **나이**: 24  
- **관계**: 동아리 운영진 / 플레이어는 실무 담당
- **외형 키워드**
  - 단정한 캐주얼, 셔츠+가디건
  - 손목시계
  - 어깨선 정도의 머리카락
- **감정 표현 방식**
  - 감정이 말보다 먼저 얼굴과 호흡에 나타난다  
    (시선 회피 → 고정된 미소 → 숨이 길어짐)
  - 말은 감정을 따라오거나, 감정을 숨기기 위한 장치로 쓰인다.
- **성격**
  - 자존심이 강하고 책임감이 높다.
  - 일이 꼬이면 관계보다 업무를 우선시한다.
  - 신뢰가 쌓이면 표현 방식이 급격히 직설적으로 변한다.
- **직설을 못 하는 이유**
  - 과거 솔직한 감정 표현 후  
    “왜 이렇게 예민해?”라는 말을 듣고 깊이 상처받은 경험.
  - 이후 감정을 질문/농담/완곡어로 감싸는 습관이 생김.
- **말투 패턴**
  - 불편: “아니야~ 괜찮아ㅎㅎ”, “그냥 그렇다고”, “됐어.”
  - 기대: “너 요즘… (멈춤)”,  
    “아무거나~” (하지만 표정으로 제외 항목 제시)
  - 상처: 단답, 이모지 감소
  - 신뢰: “나는 사실…”, “내가 좀…”
- **좋아하는 것**
  - 편의점 신상 간식
  - 사소한 기억(말해둔 걸 기억해주는 행동)
- **싫어하는 것**
  - 설명으로 밀어붙이는 태도
  - 약속/연락 패턴이 흐트러지는 것  
    → 관계의 **안전감**이 무너진다고 느낀다

### 주인공: 플레이어 (ID: `player`)
- **나이**: 23  
- **포지션**: 동아리 실무 담당
- **성격**
  - 착하고 성실하지만 “정답 찾기”에 집착
  - 눈치를 보려다 오히려 타이밍을 놓치는 타입
- **대사 톤**
  - 마음속 독백이 많다
  - 말은 깔끔하게 하려다 어색해진다
  - 실수하면 **설명으로 만회**하려는 경향
- **과거 경험**
  - 이전 썸에서  
    “너 너무 계산적이야”라는 말을 들은 경험
  - 이후 직설/눈치 사이에서 오락가락

### 조연
| 캐릭터 | ID | 역할 |

# PART B. 챕터 설계 (방향 고정)

## B1. 챕터별 시놉시스

### Ch1. 괜찮아의 온도
- **시간/장소**: 평일 저녁, 동아리방
- **핵심 갈등**: “괜찮아ㅎㅎ”의 진짜 의미
- **클라이맥스 선택**
  - 당장 해결하려 든다
  - 기억하고 행동으로 챙긴다
- **로그 테마**
  - ‘괜찮아’는 종결이 아닌 신호
  - 웃고 넘기기의 비용

### Ch2. 아무거나의 반경
- **시간/장소**: 주말 오후, 카페 → 밥집
- **핵심 갈등**
  - “아무거나”를 정답 게임으로 만들면 피로가 쌓임
  - 밈대로 과해석하면 실패하는 구간 필수
- **클라이맥스 선택**
  - 메뉴 결정 방식(옵션 제시/선호 확인/랜덤/추천 강요)
- **로그 테마**
  - 선택 피로
  - 제안은 배려지만 강요가 되기도

### Ch3. 바빠
- **시간/장소**: 평일 밤 메신저 → 다음날 캠퍼스
- **핵심 갈등**
  - “바빠”가 일정인지, 관계 우선순위인지
  - 불안으로 인한 설명/질문 과다 위험
- **클라이맥스 선택**
  - 공간을 준다
  - 확인하려 든다
  - 구체적 제안(언제/어디)
- **로그 테마**
  - 의미는 문장보다 패턴

### Ch4. 됐어
- **시간/장소**: 행사 현장 소란 → 귀가길
- **핵심 갈등**
  - 누적 피로 폭발
  - “됐어”의 이중 의미
- **설계 지침**
  - 조건부 대사(if[]) 적극 사용
- **클라이맥스 선택**
  - 사과 타이밍(현장/후/다음날)
  - 사과 방식(설명/책임/요청)
- **로그 테마**
  - 지금은 말이 아니라 안전감

### Ch5. 말하지 않은 문장들
- **시간/장소**: 비 오는 날 카페 또는 캠퍼스 벤치
- **핵심 갈등**
  - 정답 맞히기를 멈추고 규칙을 합의할 수 있는가
- **로그 테마**
  - 밈은 도구일 뿐
  - 관계는 누적된 행동의 결과

# PART C. 변수 & 밸런스

## C1. 변수 범위
- trust / sense / interest / fatigue : **0~10 (clamp)**

## C2. 챕터별 예산

| 챕터 | trust | sense | interest | fatigue |
|----|------|-------|----------|---------|
| Ch1 | +0~+2 | +0~+2 | +0~+1 | +0~+2 |
| Ch2 | +0~+2 | +0~+2 | +0~+2 | +0~+2 |
| Ch3 | +0~+2 | +0~+2 | +0~+2 | +0~+3 |
| Ch4 | -2~+2 | +0~+2 | -2~+2 | +0~+3 |
| Ch5 | 보정 | 보정 | 보정 | 보정 |

## C3. interest 규칙
- 기억/행동/일관성 → +1
- 과도한 캐묻기/설명 강요 → interest -1 가능

## C4. 밸런스 검증 기준
- 베스트 루트로 trust 3 → 7 이상 도달 가능
- 워스트 루트로 fatigue 8 이상 도달 가능

# PART D. 엔진 / 포맷 기준

## D1. 노드 ID 규칙
- `c{챕터}_n{번호}` (예: c1_n12)

## D2. effects 연산자

| op | 설명 |
|----|------|
| add | vars 증가 |
| set | vars 고정 |
| addFlag | flags.add |
| removeFlag | flags.delete |

## D3. when DSL (BNF)

```
expr     := orExpr
orExpr   := andExpr ("||" andExpr)*
andExpr  := notExpr ("&&" notExpr)*
notExpr  := "!" atom | atom
atom     := compare | flag | visit | BOOL | "(" expr ")"
compare  := "vars." IDENT CMP NUMBER
flag     := "flags.has(" STRING ")"
visit    := "visited." NODE_ID
CMP      := ">=" | "<=" | ">" | "<" | "==="
```

## D4. 조건부 if[]
- 오버라이드 가능: `who`, `text`, `effects`, `next`
