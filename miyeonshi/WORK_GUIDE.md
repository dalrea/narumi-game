# 눈치 로그 — 작업 가이드

## 현재 상태
- **기준 문서**: `girlanguage_vn_kickstarter_v3_5_full.md` (Single Source of Truth)
- **Ch1**: 완성 (`ch1/ch1.json`, `ch1/ch1_logs.json`) — 검토 통과
- **Ch2~Ch5**: 미작성
- **엔진**: 미구현
- **이전 버전**: `legacy/` 폴더에 보관 (참조 불필요)

## 작업 흐름

### 시나리오 작성 (Claude에게 요청)
1. v3.5 문서를 읽게 한다
2. 챕터 JSON + 로그 JSON을 작성 요청한다
3. 검토를 요청하면 아래 기준으로 피드백을 받는다:
   - 스펙 준수 (노드 수, 선택지, if[], nextIf, 로그, 변수 예산)
   - 재미/몰입 (감정 곡선, 분기 후속, 캐릭터 활용)
   - 밸런스 (변수 도달 가능성, 예산 정합성)
   - 로그 품질 (장면 특화 메타 코멘트 여부)
4. 피드백 반영 후 재검토 → 통과할 때까지 반복

### 검토 요청 프롬프트 예시
```
miyeonshi 폴더 안에 기획 문서(v3.5)와 ch1 시나리오가 있어.
v3.5를 기준으로 ch{N}을 작성했는데 검토 부탁해.
스펙 준수, 재미, 밸런스, 로그 품질 전부 봐줘.
```

### 기획 문서 수정 시
- v3.5 본문을 직접 수정하거나, 패치 문서를 만든 뒤 병합
- **절대 처음부터 재작성하지 않는다** (누락 반복 방지)

## 스펙 체크리스트 (챕터당)

| 항목 | 기준 |
|------|------|
| 노드 수 | 25~30 (연출 노드 제외 기준) |
| 선택지 | 4~7회 |
| 조건부 say (if[]) | ≥ 4 |
| nextIf | ≥ 2 |
| 로그 entries | ≥ 8 |
| interest 변동 | ≥ 1회 |
| fatigue 예산 | Ch1~3: +0~+2, Ch4: +0~+3 |
